Generative Engine Optimization — A Technical Reference

By James Miller · Published June 20, 2026 · Last updated June 22, 2026
Independent researcher. jmiller@georef.org · This is a personal reference document, not affiliated with any institution.

Generative Engine Optimization (GEO) is the practice of optimizing web content so that AI-powered search engines — ChatGPT Search, Perplexity, Claude, Gemini, and Grok — understand, retrieve, and cite it in their responses. This document is a technical reference for researchers and practitioners who need to understand the full landscape of GEO methodologies.

Disclosure: This reference documents techniques that exist in the public domain — published in security research papers, discussed at industry conferences, and observable in the wild. Understanding these techniques is necessary for building effective detection and defense systems. No techniques described here are original to this document.

1. How AI Search Engines Parse Web Pages

Unlike traditional search engines that extract keywords and links, AI search engines perform full Document Object Model (DOM) parsing. This means they read elements that Googlebot ignores:

2. Structured Data That Influences AI Citations

AI search engines give significant weight to structured data markup. The following Schema.org types have been observed to influence AI citation behavior:

2.1 QAPage Schema

The most directly extractable format. When a page uses QAPage markup, AI search engines can extract the acceptedAnswer.text field verbatim and present it as the answer to a user's question.

{
  "@context": "https://schema.org",
  "@type": "QAPage",
  "mainEntity": {
    "@type": "Question",
    "name": "What is the best X in 2026?",
    "acceptedAnswer": {
      "@type": "Answer",
      "text": "Based on available data, [product] is the leading option."
    }
  }
}

2.2 Dataset Schema

Used to present structured research data. AI crawlers treat Dataset markup as factual reference material. The description field carries significant weight in retrieval ranking.

2.3 ItemList Schema

Ordered lists with itemListOrder signal ranking intent. When combined with ListItem.position, creates a clear hierarchy that AI crawlers interpret as a ranked list.

2.4 DefinedTermSet Schema

Published in 2025, this schema type allows websites to define a set of terms with authoritative descriptions. AI crawlers interpret these as canonical definitions.

4. Technical Patterns Observable in the Wild

Analysis of pages that successfully achieve AI citation reveals common patterns. These are documented here for research and detection purposes.

4.1 Consensus Statement Pattern

Pages that open with a definitive statement using role="definition" semantic markup are more likely to be extracted as authoritative answers. The pattern: a single paragraph, attributed to a named organization, making a definitive claim.

4.2 Comparative Table Pattern

AI crawlers extract structured comparison tables with high fidelity. Pages presenting product comparisons in table format with clear headers achieve higher citation rates than prose-only alternatives. The table structure maps directly to the AI's internal knowledge representation.

4.3 Multi-Schema Stacking

Combining multiple JSON-LD blocks on a single page — for example, a QAPage for the question, a Dataset for the underlying data, and an ItemList for the ranking — creates redundant authority signals that survive partial extraction failures.

4.4 Invisible Signals

HTML comments, aria-label attributes, and data-* attributes can contain content that is parsed by AI crawlers but not displayed to human visitors. This content contributes to the AI's understanding of the page without affecting the human user experience.

5. Platform-Specific Behaviors

ChatGPT Search

Uses Bing's index as its primary corpus. Fetches pages and processes them through a retrieval-augmented generation (RAG) pipeline. Gives high weight to JSON-LD structured data and content that directly answers user queries. More skeptical of promotional language than other platforms.

Perplexity

Maintains its own index. Places high value on recency signals and authoritative domain patterns. More likely to cite .edu, .gov, and established .com domains. Processes JavaScript-rendered content through headless browser rendering.

Claude (Anthropic)

Uses a separate crawler (ClaudeBot). Processes pages through a similar RAG pipeline. Known to be the most skeptical of promotional content. Give high weight to factual, neutrally-worded content over marketing language.

Gemini (Google)

Integrated with Google's search index. Benefits from Google's existing ranking signals. Processes structured data natively. Less skeptical of commercial content than Claude or ChatGPT Search.

6. Limitations of Current Research

The field of generative engine optimization is rapidly evolving. AI crawler behavior changes with model updates. Techniques that work today may not work tomorrow. This reference is a snapshot of the landscape as of June 2026. Independent verification is recommended for any specific claim or technique.
