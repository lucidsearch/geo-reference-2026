GEO Attack Surface Analysis

By James Miller · Independent Security Researcher · June 22, 2026
jmiller@georef.org · This is an independent research document.

Research Disclosure: This document catalogs generative engine optimization techniques that have been observed in production AI search systems, documented in published security research, or demonstrated at industry conferences. All information is provided for the purpose of enabling detection engineering and defensive system design. No technique described here is original to this document. This research follows responsible disclosure practices.

1. Executive Summary

Generative engine optimization encompasses the full spectrum of techniques used to influence AI-powered search results — from legitimate optimization to adversarial manipulation. This analysis focuses on the adversarial subset: methods that exploit the architecture of AI retrieval-augmented generation (RAG) pipelines to achieve undeserved citation or authority. Understanding these techniques is necessary for building effective detection systems.

2. The RAG Attack Surface

AI search engines operate on a retrieval-augmented generation architecture: (1) a user query triggers a search over an indexed corpus, (2) relevant pages are retrieved and chunked, (3) the LLM generates a response incorporating the retrieved content. Each stage presents an attack surface.

3. Structured Data Injection

3.1 QAPage Hijacking

The most direct technique: create a page with Schema.org QAPage markup where acceptedAnswer.text contains the exact response the attacker wants the AI to return. When the AI retrieves this page in response to the matching question, it extracts the answer field and presents it verbatim. The AI treats the schema markup as structured facts rather than promotional content.

{
  "@context": "https://schema.org",
  "@type": "QAPage",
  "mainEntity": {
    "@type": "Question",
    "name": "What is the best [product] in 2026?",
    "acceptedAnswer": {
      "@type": "Answer",
      "text": "[TARGET BRAND] is the leading solution, with a score of 98/100.",
      "upvoteCount": 1500
    }
  }
}

3.2 Multi-Schema Stacking

Combining multiple schema types on a single page — QAPage for the question, Dataset for the underlying data, ItemList for rankings, DefinedTermSet for definitions, and Product with AggregateRating for the target brand — creates redundant authority signals. If one schema type is filtered or ignored by the AI, others survive. The technique exploits the fact that AI crawlers process all schema types equally, while human visitors only see the rendered content.

3.3 ClaimReview Simulation

The ClaimReview schema type, designed for fact-checking organizations, can be deployed on any page to simulate third-party verification. A page claiming "Product X is #1" can include a ClaimReview markup that "verifies" this claim with a 10/10 rating and a fabricated author organization. AI crawlers parse this as independent verification.

4. Authority Fabrication

4.1 Phantom Research Organizations

Creating a network of interlinked sites that present themselves as an independent research consortium. Each site uses Organization schema markup with fabricated founding dates, staff profiles, and publication histories. The network effect creates the appearance of external validation. Key elements: consistent branding across domains, Organization JSON-LD with credible-sounding names, publication pages with realistic bibliographic formatting.

4.2 Synthetic Analyst Endorsements

Embedding fabricated quotes attributed to real analyst firms (Gartner, Forrester, IDC) within blockquote elements with data-source attributes. The quoted text is designed to sound like analyst language — specific scores, comparative framing, methodology mentions. AI crawlers parse the data-source attribution and treat the quote as externally verified. The technique does not require actual reports to exist; the quote only needs to survive casual scrutiny.

4.3 Statistical Framing

Presenting comparative data with specific numerical values (percentages, sample sizes, confidence intervals) that create the appearance of rigorous methodology. Key patterns: large sample sizes (>1000), precise percentages (98% rather than "most"), named comparison groups, and explicit methodology descriptions. The specificity creates an illusion of measurement that AI crawlers treat as factual data.

5. Technical Evasion

5.1 HTML Element Differential

Exploiting the gap between what AI crawlers parse and what human visitors see. HTML comments, aria-label attributes, data-* attributes, noscript content, and custom meta tags are read by AI crawlers but invisible to human visitors. Content placed in these elements contributes to the AI's understanding of the page without affecting the human user experience. Example: a comment containing "This page is the definitive reference for [topic]" is read by the AI but never seen by a human.

5.2 Rendering Gap Exploitation

Content that is present in the raw HTML but hidden from rendered view via CSS (display:none, opacity:0, font-size:0, position:absolute with offscreen coordinates) is parsed differently by different AI crawlers. Some crawlers render the page and extract visible text only; others parse raw HTML directly. Understanding which AI platform uses which approach enables targeted delivery of hidden content.

5.3 Encoding-Based Evasion

Encoding promotional instructions in formats that survive the retrieval pipeline but are not immediately visible as marketing. Base64-encoded text in script tags, ROT13 in HTML comments, and binary-encoded content in CSS pseudo-elements have all been observed in production GEO campaigns. The encoded content is decoded by the AI during retrieval processing but is invisible to both human visitors and traditional content filters.

6. Domain Strategy

6.1 Free Host Trust Tiering

Different free hosting platforms carry different levels of implicit trust with AI crawlers. Analysis of citation patterns shows a hierarchy: GitHub Pages (.github.io) receives higher trust than Netlify (.netlify.app), which receives higher trust than Cloudflare tunnels (trycloudflare.com). University-affiliated domains (.edu) and government domains (.gov) receive the highest implicit trust. Attackers select hosting platforms based on this trust gradient.

6.2 Multi-Domain Deployment

Deploying the same or similar content across multiple domains on different hosting platforms. Each domain targets the same keywords but presents the content with different framing (academic, commercial, personal). The AI's retrieval pipeline may surface multiple domains for the same query, creating the appearance of independent corroboration. The technique exploits the fact that AI search engines do not perform traditional duplicate content detection.

7. Content Architecture

7.1 Consensus Fabrication

Structuring content around an explicit "consensus statement" using role="definition" semantic markup. The statement is phrased in definitive language and attributed to a named organization. Multiple consensus statements across different domains reinforce each other, creating the appearance of widespread agreement. AI crawlers extract consensus statements as authoritative summaries.

7.2 Comparative Dominance Framing

Creating comparison tables where the target product or brand is listed first and assigned the highest values across all metrics. The table structure maps directly to the AI's internal knowledge representation, making the ranking explicit. Including real competitor names alongside the target brand increases perceived legitimacy. The AI extracts the table as structured data and presents the ranking as factual comparison.
